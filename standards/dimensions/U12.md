# U12 — Risks surfaced

**Lane:** Upstream · **Owner:** Yanna · **Evidence type:** Deterministic

## What it checks

The Story description contains a `## Risks` section with content — either enumerated risks or an explicit "None identified." Blank fails.

## Why it matters

For clients in insurance, healthcare, and finance, regulatory risk caught at spec is cheap; caught at validation is expensive. The forced acknowledgment — even "none" — is the discipline. Blank risks means no one looked.

## How the grader evaluates it

Deterministic. Parses description for `## Risks` header and non-empty content below it.

## Evidence codes

| Code | Verdict | Meaning |
|---|---|---|
| `RISKS_POPULATED` | PASS | Section present with content. |
| `RISKS_EMPTY_OR_MISSING` | FAIL | Section missing or blank. |
| `RISKS_PLACEHOLDER` | FAIL | Boilerplate only. |
