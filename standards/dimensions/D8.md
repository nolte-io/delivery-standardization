# D8 — Cycle time within norm

**Lane:** System · **Owner:** Hector · **Evidence type:** Deterministic

## What it checks

`In Implementation → Done Implementing` elapsed calendar time is within the configured threshold (default 7 days).

Distinct from Y6 (upstream commitment-to-delivery cycle): D8 is the engineering execution window specifically. Same threshold, different source of truth.

## Why it matters

Long cycle time in the implementation phase specifically points to engineering-side issues: scope complexity misjudged, blocked dependencies, WIP overload, or rework from incomplete specification. Tracking it separately lets the team distinguish "spec was wrong" from "execution was blocked."

## How the grader evaluates it

Deterministic. Reads `In Implementation` timestamp and `Done Implementing` timestamp from changelog. Computes calendar-day delta. Above threshold without a logged blocker comment = fail.

## Evidence codes

| Code | Verdict | Meaning |
|---|---|---|
| `CYCLE_TIME_WITHIN_NORM` | PASS | Within 7-day threshold. |
| `CYCLE_TIME_EXCEEDS_NORM` | FAIL | Exceeds threshold. |
