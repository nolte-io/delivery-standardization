# D10 — No backward transitions

**Lane:** System · **Owner:** Hector + Yanna · **Evidence type:** Deterministic

## What it checks

The Story's changelog contains only forward transitions through the kanban order. Any regression (e.g., `In Validation → In Implementation`) fails.

## Why it matters

Backward transitions are silent rework. They hide defects, hide scope evolution, and hide the fact that validation failed — because none of it gets logged as a Story Defect. The correct response to a failed validation is a Story Defect, not a status revert. D10 exists to surface this explicitly.

## How the grader evaluates it

Deterministic. Walks the Story's status history. Any transition that moves backward in the configured kanban order = fail.

## Evidence codes

| Code | Verdict | Meaning |
|---|---|---|
| `FORWARD_ONLY_TRANSITIONS` | PASS | All status moves forward. |
| `BACKWARD_TRANSITION_DETECTED` | FAIL | At least one regression. |
