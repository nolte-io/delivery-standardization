# D5 — Tests passing with evidence

**Lane:** Downstream · **Owner:** Hector · **Evidence type:** Deterministic

## What it checks

Every Story has a sub-task whose title matches `Tests — [Story key]`, closed before `Done Implementing`, with a CI run link in the description.

## Why it matters

"Tests passing" without evidence is an assertion, not a fact. The convention exists so that validation, audit, and future debugging have a single anchor for "how do we know it worked." Without it, "works on my machine" becomes the implicit standard, and regressions compound silently.

## How the grader evaluates it

Deterministic. Parses sub-tasks for the title pattern. Checks status and description content at the Done Implementing transition.

## Evidence codes

| Code | Verdict | Meaning |
|---|---|---|
| `TEST_SUBTASK_CLOSED_WITH_CI_LINK` | PASS | Convention met. |
| `TEST_SUBTASK_MISSING` | FAIL | No matching sub-task. |
| `TEST_SUBTASK_OPEN` | FAIL | Sub-task present but open. |
| `TEST_SUBTASK_MISSING_CI_LINK` | FAIL | Sub-task closed but no CI link. |
