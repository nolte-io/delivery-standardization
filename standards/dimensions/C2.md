# C2 — BDD quality at commitment

**Lane:** Upstream · **Owner:** Yanna · **Evidence type:** Judge

## What it checks

At commitment time, the `## Scenarios` section contains at least one Given/When/Then that exercises the highest-value rule implied by the AC. Not exhaustive coverage — core coverage.

## Why it matters

BDD scenarios that only cover edge cases leave the core behavior undemonstrated. A delivery can pass its tests and still fail its purpose. This dimension ensures the central rule is captured before implementation starts.

## How the grader evaluates it

Judge. Claude reads the AC and the Scenarios section as they existed at commitment time (historical snapshot via changelog), and determines whether the scenarios cover the core rule.

Judge prompt: [`../../prompts/C2.md`](../../prompts/C2.md).

## Evidence codes

| Code | Verdict | Meaning |
|---|---|---|
| `SCENARIOS_COVER_CORE_RULE` | PASS | Core behavior demonstrated. |
| `SCENARIOS_COVER_ONLY_EDGE_CASES` | FAIL | Edge cases but no core. |
| `SCENARIOS_PLACEHOLDER_OR_EMPTY` | FAIL | Empty or boilerplate. |
| `SCENARIOS_NOT_GIVEN_WHEN_THEN` | FAIL | Prose instead of GWT. |
