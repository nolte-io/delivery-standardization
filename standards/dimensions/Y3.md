# Y3 — AC in plain language, pre-implementation

**Lane:** Upstream · **Owner:** Yanna · **Evidence type:** Deterministic + Judge

## What it checks

Acceptance criteria are present before commitment, written in plain language a non-engineer can verify.

## Why it matters

AC written after the fact is retrospective rationalization, not a contract. Without AC before implementation, there is no basis for validation, no defense against scope evolution, and no shared understanding of "done."

## How the grader evaluates it

- **Y3.a (deterministic):** `## Acceptance Criteria` section populated before the commitment timestamp (Done Specifying → In Implementation). Verified via description changelog.
- **Y3.b (judge):** Claude confirms the AC is plain language, not a technical restatement.

Judge prompt: [`../../prompts/Y3b.md`](../../prompts/Y3b.md).
