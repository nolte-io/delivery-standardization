# D7 — WIP respected

**Lane:** System · **Owner:** Hector · **Evidence type:** Deterministic

## What it checks

For each day a Story was in `In Implementation`, the total concurrent count of In-Implementation Stories stayed within the configured WIP limits — per engineer and system-wide.

## Why it matters

WIP limits are the mechanism that produces predictable cycle time. Exceeded WIP means context switching, delayed feedback, and silent queuing — all of which degrade the forecast. Chronic violations trigger a system review per Rule 4.

## How the grader evaluates it

Deterministic. Reconstructs daily In-Implementation state from the changelog. Counts per-assignee and system-wide. Compares against configured limits. Exceptions recognized only when an authorized approver logs a comment containing `WIP exception:` during the window.

## Evidence codes

| Code | Verdict | Meaning |
|---|---|---|
| `WIP_WITHIN_LIMITS` | PASS | Every day within limits. |
| `WIP_EXCEEDED_PER_ENGINEER` | FAIL | A single engineer over limit. |
| `WIP_EXCEEDED_SYSTEM` | FAIL | System-wide concurrent count over limit. |
