# D6 — Operationally shippable

**Lane:** Downstream · **Owner:** Hector · **Evidence type:** Deterministic

## What it checks

At Done Implementing, the `Production Release Reference` custom field is populated with a deploy URL, release tag, or merged PR link. Same field as Y5 — two uses: Y5 confirms the reference exists at Done (validation), D6 confirms it exists at Done Implementing (engineering handoff).

## Why it matters

"Done Implementing" means ready to ship. Without a release reference, there is no evidence the work can actually be shipped — and no anchor for rollback, incident response, or release communication. Local-only or staging-only references fail.

## How the grader evaluates it

Deterministic. Reads `Production Release Reference` custom field at the Done Implementing transition.

## Evidence codes

| Code | Verdict | Meaning |
|---|---|---|
| `DEPLOY_REFERENCE_PRESENT` | PASS | Valid reference at Done Implementing. |
| `DEPLOY_REFERENCE_EMPTY` | FAIL | Field empty. |
| `DEPLOY_REFERENCE_LOCAL_ONLY` | FAIL | Reference is a local/dev branch. |
