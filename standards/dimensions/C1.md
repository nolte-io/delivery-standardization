# C1 — Commitment transition exists

**Lane:** System · **Evidence type:** Deterministic

## What it checks

The Story's changelog contains a transition from the configured commitment-gate source (`Done Specifying`) to the configured destination (`In Implementation`).

## Why it matters

Every downstream dimension — cycle time, WIP, scope evolution, sub-task discipline — anchors on the commitment timestamp. If a Story skipped the gate, those checks cannot run accurately. This dimension is the pre-flight check that makes the rest of the grader meaningful.

## How the grader evaluates it

Deterministic. Scans the changelog for the configured commitment transition. Absent = fail.

## Evidence codes

| Code | Verdict | Meaning |
|---|---|---|
| `COMMITMENT_TRANSITION_FOUND` | PASS | Transition recorded in changelog. |
| `COMMITMENT_TRANSITION_MISSING` | FAIL | No such transition found. |
