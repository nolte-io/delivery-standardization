# Y4 — Non-builder can validate

**Lane:** Upstream · **Owner:** Yanna · **Evidence type:** Deterministic

## What it checks

The account that moves a Story into Done via validation is not the same account that built it (held sub-tasks during In Implementation).

## Why it matters

Validation by the builder is not validation. A second set of eyes enforces the contract in the AC, catches obvious defects, and prevents self-approval from becoming a shortcut. For a one-person team, this dimension is flagged as a `TEAM_SIZE_EXCEPTION` via config.

## How the grader evaluates it

Deterministic. Changelog shows the accountId that executed `In Validation → Done`. Compared against sub-task assignees during In Implementation. Match = fail.

## Evidence codes

| Code | Verdict | Meaning |
|---|---|---|
| `VALIDATOR_INDEPENDENT` | PASS | Validator and builder are different accounts. |
| `VALIDATOR_IS_BUILDER` | FAIL | Same account. |
| `TEAM_SIZE_EXCEPTION` | Flag | Team size == 1 per config; skipped. |
