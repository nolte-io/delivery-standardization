# Y5 — Live in production when called Done

**Lane:** Upstream · **Owner:** Yanna · **Evidence type:** Deterministic

## What it checks

At the moment a Story transitions to Done, the Production Release Reference field is populated with a non-staging reference.

## Why it matters

A delivery is live in production or it is not a delivery. Staging does not count. Demos do not count. This dimension is the enforcement mechanism for the most load-bearing word in nolte.io/delivery: "in production."

## How the grader evaluates it

Deterministic. Reads the `Production Release Reference` custom field at the Done transition timestamp via changelog snapshot. Empty or post-Done population = fail.

## Evidence codes

| Code | Verdict | Meaning |
|---|---|---|
| `PRODUCTION_REFERENCE_PRESENT_AT_DONE` | PASS | Non-staging reference recorded at Done. |
| `PRODUCTION_REFERENCE_EMPTY` | FAIL | Field empty. |
| `PRODUCTION_REFERENCE_STAGING_ONLY` | FAIL | Reference is staging URL. |
| `PRODUCTION_REFERENCE_POST_DONE` | FAIL | Populated after Done transition. |
