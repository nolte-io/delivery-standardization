# D3 — Violations surfaced, not absorbed

**Version:** 0.1
**Type:** Judge (J)
**Lane:** Downstream
**Owner:** Hector

## Purpose

Detect silent deviation from agreed scenarios during implementation.

**Known limitation:** This dimension asks the judge to detect a negative (silent absorption). Expect higher noise than other dimensions. Candidate for promotion to `claude-opus-4-7` via `judge.model_by_dimension` if Sonnet's first-run accuracy is insufficient.

## Inputs

- `{{story_key}}`
- `{{scenarios}}` — agreed scenarios at commitment
- `{{comments_concatenated}}` — all comments on the Story during `In Implementation`
- `{{subtasks_text}}` — titles and descriptions of all sub-tasks

## Evidence codes

| Code | Verdict |
|---|---|
| `NO_DEVIATION_DETECTED` | PASS |
| `DEVIATION_SURFACED_EXPLICITLY` | PASS |
| `DEVIATION_ABSORBED_SILENTLY` | FAIL |
| `EVIDENCE_INSUFFICIENT_TO_JUDGE` | INSUFFICIENT_EVIDENCE |

## User prompt

```
Dimension: D3 — Violations surfaced, not absorbed.

Nolte's rule: if execution would deviate from agreed scenarios, stop and surface it. Silent deviation — changing behavior without raising it, finding a workaround without flagging it, absorbing rework without comment — is a system failure.

Grade the evidence:

PASS (NO_DEVIATION_DETECTED): comments and sub-tasks show execution of the agreed scenarios without contradiction. No signal of rework or scope shift.

PASS (DEVIATION_SURFACED_EXPLICITLY): comments or sub-tasks contain a visible flag that scenarios could not be executed as agreed — and the flag names what changed and why.

FAIL (DEVIATION_ABSORBED_SILENTLY): comments or sub-tasks contain language suggesting the delivered behavior differs from the scenarios, but no flag, escalation, or scope discussion appears. Look for language like "ended up doing X instead," "switched approach," "had to change," "this turned out to be Y" without a corresponding earlier flag.
```
