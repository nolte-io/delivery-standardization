# C2 — BDD quality at commitment

**Version:** 0.1
**Type:** Judge (J)
**Lane:** Upstream
**Owner:** Yanna

## Purpose

Confirm the scenarios at commitment time cover the highest-value rules, not only trivial edge cases.

## Inputs

- `{{story_key}}`
- `{{acceptance_criteria_at_commit}}` — AC text snapshot at commitment timestamp
- `{{scenarios_at_commit}}` — Scenarios text snapshot at commitment timestamp

## Evidence codes

| Code | Verdict |
|---|---|
| `SCENARIOS_COVER_CORE_RULE` | PASS |
| `SCENARIOS_COVER_ONLY_EDGE_CASES` | FAIL |
| `SCENARIOS_PLACEHOLDER_OR_EMPTY` | FAIL |
| `SCENARIOS_NOT_GIVEN_WHEN_THEN` | FAIL |

## User prompt

```
Dimension: C2 — BDD quality at commitment.

Passing scenarios: contain at least one Given/When/Then that exercises the highest-value rule implied by the acceptance criteria. The core behavior — the thing the delivery is for — is demonstrated.

Failing scenarios: are empty or placeholder; are written as prose instead of Given/When/Then; cover only trivial edge cases (error states, validation, boundary conditions) while leaving the core rule undemonstrated.

You are judging coverage of the core rule, not exhaustive coverage. Nolte's rule is "highest value rules/scenarios, not every rule." One scenario that exercises the core behavior is passing.

Story: {{story_key}}
Acceptance Criteria at commitment:
---
{{acceptance_criteria_at_commit}}
---

Scenarios at commitment:
---
{{scenarios_at_commit}}
---

Grade C2. Return JSON only.
```
