# D2 — Design artifact necessity and presence

**Version:** 0.1
**Type:** Judge (J) + Deterministic (D)
**Lane:** Downstream
**Owner:** Hector

## Purpose

Determine whether the work warrants a design artifact and whether one exists.

## Inputs

- `{{story_key}}`
- `{{story_title}}`
- `{{story_description}}`
- `{{acceptance_criteria}}`
- `{{scenarios}}`
- `{{design_artifact_present}}` — boolean, whether the Design Artifact Link field is populated

## Evidence codes

| Code | Verdict |
|---|---|
| `DESIGN_NOT_REQUIRED_WORK_TRIVIAL` | PASS |
| `DESIGN_PRESENT_AS_REQUIRED` | PASS |
| `DESIGN_MISSING_FOR_COMPLEX_WORK` | FAIL |

## Complexity heuristics

Work requires a design artifact if it involves:
- Cross-system integration.
- New data model or schema change.
- New UI flow crossing multiple screens.
- Regulatory implication (insurance, healthcare, finance).
- Novel architecture or unproven approach.

Work does not require one if it is a small bugfix, config change, copy change, dependency bump, or narrow change inside an existing pattern.
